食分好找 (Foodsaver) - 跨超商即期品資訊整合平台
📖 專案簡介
在台灣，即期品資訊往往散落在各個超商的應用程式或平台中，造成使用者查詢耗時且難以規劃 。本專案「食分好找 (Foodsaver)」旨在打造一個以「省錢省時」與「減少食物浪費」為主軸的即期品資訊網站 。透過模擬真實超商即期品資料，將零散資訊轉化為直觀的互動體驗，讓惜食成為輕鬆實踐的生活選項 。



✨ 核心特色
本系統以「跨超商整合 + 個人化系統」為核心概念，具備以下主要功能 ：


跨商家整合資訊：突破單一品牌限制，整合 7-Eleven、全家、OK、萊爾富等不同品牌的即期品資訊 。

地圖視覺化呈現：

以互動地圖顯示周邊店家，圓圈大小代表剩餘數量，不同顏色區分不同超商品牌 。


整合 Nominatim API，支援地標快速轉換為座標並顯示周邊資訊 。


AI 智慧助理：內建 AI 問答功能（串接 LLaMA 模型），能回答如「今天吃什麼？」等問題，並依據即時庫存提供建議 。



個人化會員體驗：


喜好清單：會員可建立清單，系統會主動推送折扣通知 。




數據洞察：自動彙整使用者偏好數據，透過視覺化圖表呈現採買習慣與趨勢 。

精準搜尋與導航：

支援依名稱、類別、價格區間與剩餘數量篩選商品 。

點選店家後可直接串接 Google Map 進行導航 。



雙模式首頁：區分「訪客模式」（快速瀏覽推薦清單）與「會員模式」（專屬喜好推薦與通知） 。

🛠️ 開發技術與環境
本系統使用 Python 語言開發，開發環境為 Visual Studio Code (VSCode) 。


後端框架：Flask (輕量、快速、易部署) 。


前端技術：HTML + JavaScript，整合 OpenStreetMap API、Nominatim API 及 Google Map 。


資料庫：SQLite (儲存商品資訊、使用者喜好、歷史紀錄與模擬資料) 。


AI 技術：使用 OpenAI 套件串接 OpenRouter.ai 平台的 LLaMA 模型 API 。


運行環境：桌面/筆電瀏覽器為主，需網路連線以支援即時功能 。

👥 目標客群

一般消費者（特別是學生）：節省支出並快速找到沿途的折扣商品 。


注重永續環保族群：希望減少剩食的使用者 。


價格敏感型消費者：善用科技工具規劃採買以獲得優惠的族群 。

🚀 未來展望

拓展至量販店與超市 。

透過 AI 強化需求預測與推薦功能，結合大數據分析發展全方位智慧採買助理 。
